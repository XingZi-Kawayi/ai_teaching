# 3、Text Splitter——文本切割（分块）

我们在使用 AI 大语言模型时，核心痛点之一是**模型的上下文窗口有限**，比如想让 AI 回答某本书的问题，但整本书的文字量远超模型能接收的最大长度，没法直接把整本书塞给 AI。解决这个问题的关键就是把长文档拆成一个个 AI 能理解、长度也符合要求的文本块，而负责这个拆分工作的工具就是**文本分割器**，今天我们就结合 LangChain 的代码，手把手讲清楚怎么用适合中文的文本分割器处理长文档，全程不改动核心代码，保证实操性。

## 核心前提：分割文本的关键要求

拆分文本不是随便切分就行，核心是**不能切出半句话、三分之一句话**，如果分割后的文本块语义不完整，我们看不懂，AI 更没法理解；另外还要兼顾文本的上下文连贯性，避免相邻文本块之间丢失关键关联信息。

LangChain 中处理文本分割最常用的是**字符递归分割器（RecursiveCharacterTextSplitter）**，这个分割器可以自定义切分的符号和优先级，特别适合适配中文的语言习惯，接下来我们从安装到代码实操一步步讲。

## 第一步：安装分割器对应的依赖库

字符递归分割器不在 LangChain 的核心库中，而是在langchain_text_splitters这个独立库里，所以第一步需要先安装这个依赖：

```bash
pip install langchain_text_splitters
```

## 第二步：导入所需的工具模块

安装完成后，我们需要导入两个核心模块：一个是**加载文档的模块**，一个是**文本分割器模块**，对应代码如下：

```python
from langchain_community.document_loaders import TextLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
```

- **TextLoader**：LangChain 中专门用来加载 **纯文本文件（.txt）** 的工具，后续我们用它加载本地的长文档；
- **RecursiveCharacterTextSplitter**：我们核心要用的**字符递归分割器**，后续创建实例来实现文本分割。

## 第三步：加载本地需要处理的长文档

用TextLoader加载本地的 demo.txt 文件（你可以替换成自己的长文档路径），代码如下：

```python
loader = TextLoader("./demo.txt")
docs = loader.load()
```

- 第一行：创建文档加载器实例，指定要加载的文件路径（./表示当前目录）；
- 第二行：调用load()方法执行加载，得到的docs是**文档对象组成的列表**，这个对象里包含了文档的文本内容（page_content）、文件路径等信息，后续我们就是对docs里的文本内容进行分割。

## 第四步：创建字符递归分割器实例（核心参数配置）

这是整个实操的核心步骤，我们需要给分割器配置**文本块长度、重叠长度、自定义分隔符**三个关键参数：

```python
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=40,
    separators=["\n\n", "\n", "。", "！", "？", "，", "、", ""]
)
```

下面逐个讲清楚每个参数的作用，以及为什么要这么配置（尤其是适配中文的分隔符）：

### 1. chunk_size：文本块的最大长度

chunk_size=500表示**每个分割后的文本块，最大字符长度为 500**（这里的长度按字符数计算）。这个数值是为了演示设置的较小值，实际使用时可以根据你用的大模型上下文窗口调整（比如模型支持 2000 字符的上下文，就可以设为 1500 左右），核心是让每个文本块的长度不超过模型能接收的范围。

### 2. chunk_overlap：相邻文本块的重叠长度

chunk_overlap=40表示**两个相邻的文本块，会有 40 个字符的内容重叠**。这一步是为了**保持文本的上下文连贯性**：如果完全没有重叠，分割线的位置可能刚好是关键信息的交界处，导致两个文本块丢失关联；而少量的重叠能让 AI 看到相邻文本块的衔接信息，比如上一个块的结尾和下一个块的开头重叠，语义就完整了。

### 3. separators：自定义分割符列表（重点适配中文）

separators是**分割符的优先级列表**，这是字符递归分割器的核心设计，也是适配中文的关键，我们先讲它的**工作逻辑**，再讲中文的配置原因：

#### 分割器的递归工作逻辑

分割时会**从列表第一个分隔符开始尝试切分**，如果按当前分隔符切出的文本块，长度仍然超过chunk_size的 500，就会用列表里的**下一个分隔符**继续切分；如果还是超长度，就继续往下试，直到切出的文本块长度符合要求，最后实在没招了，就用空字符串（任意位置切分）。

#### 中文的分隔符配置原因

LangChain 的分割器有**默认的分隔符列表**，但默认列表里包含空格，而**中文的书写习惯几乎不用空格分隔**，默认配置切分中文会很生硬，甚至切出半句话，所以我们需要**自定义适合中文的分隔符优先级**，本次配置的列表：

```python
["\n\n", "\n", "。", "！", "？", "，", "、", ""]
```

优先级从高到低，完全按中文的语义单元切分：

- `\n\n`：两个换行符，对应文档中**不同段落的分隔**，先按大的语义单元（段落）切分，这是最理想的切分方式，语义最完整；
- `\n`：一个换行符，对应段落内的换行，比段落低一级的语义单元；
- `。`、`！`、`？`：中文里**句子的结束符号**，如果按换行切分的块还是太长，就按完整句子切分，保证每个块至少是完整的句子；
- `，`、`、`：中文里**句中的分隔符号**，如果完整句子还是太长，就按分句切分，尽量保证语义的完整性；
- `""`：空字符串，最后兜底的方式，任意位置切分，只有前面所有分隔符切分后仍超长度时才会用。

## 第五步：执行文本分割，得到分割后的文本块

创建好分割器实例后，调用split_documents方法，传入我们之前加载的文档列表docs，就能完成分割，代码如下：

```python
texts = text_splitter.split_documents(docs)
```

- **方法作用**：split_documents是专门处理文档对象列表的分割方法，会自动提取docs中每个文档对象的page_content（文本内容）进行分割；
- **分割结果**：texts仍然是**文档对象组成的列表**，和原docs的格式一致，但**每个对象的page_content长度都不会超过我们设置的 500**，也就是我们需要的、长度符合要求的文本块。

## 第六步：查看分割后的效果

最后我们可以打印出第一个文本块的内容，看看实际的分割效果，代码如下：

```python
print(texts[0].page_content)
```

- texts[0]：取分割后文本块列表的第一个对象；
- .page_content：提取该对象的文本内容（这是我们最终要传给 AI 的内容）；
- 执行这行代码，就能在控制台看到第一个文本块的具体文字，验证分割是否符合预期。

## 完整代码回顾

把上面的步骤整合起来，就是可直接运行的完整代码，复制到 Python 环境中，替换./demo.txt为自己的文档路径，就能实现中文长文档的分割：

```python
# 安装依赖
# pip install langchain_text_splitters

from langchain_community.document_loaders import TextLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter

# 加载文档
loader = TextLoader("./demo.txt")
docs = loader.load()

# 创建分割器
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=40,
    separators=["\n\n", "\n", "。", "！", "？", "，", "、", ""]
)

# 执行分割
texts = text_splitter.split_documents(docs)

# 查看结果
print(texts[0].page_content)
```

## 文本分割后的下一步

我们把长文档分割成一个个符合长度要求的文本块后，**下一步就是把这些文本块转换成向量**（这个过程叫**嵌入**），后续就可以通过向量检索，把和用户问题相关的文本块找出来，再传给大语言模型，让 AI 基于相关文本块回答问题，这也是处理大语言模型长文档的核心流程。
